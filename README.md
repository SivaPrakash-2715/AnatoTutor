# 🫀 AnatoTutor

**Multimodal RAG-Based Anatomy Learning System**

AnatoTutor is an AI-powered educational assistant that answers anatomy questions using content retrieved strictly from trusted medical textbooks. It combines semantic vector search (FAISS), image captioning (BLIP), and large language model generation (Llama 3.1 via Groq) to deliver structured, hallucination-free explanations — with both text and image query support.

> Built as a B.Tech final-year capstone project at University College of Engineering, Anna University Tiruchirappalli.

---

## ✨ Features

- **Multimodal input** — accepts typed anatomy questions or uploaded images (X-rays, diagrams, specimen photos)
- **RAG-constrained generation** — LLM answers only from retrieved textbook chunks; no hallucinated content
- **Structured academic output** — every response follows a consistent format: Definition → Structure → Function → Clinical Relevance → Key Points → Memory Tip
- **Marks-aware depth control** — specify marks (2 / 5 / 10) and the response length adjusts automatically
- **Image-to-text retrieval** — uploaded images are captioned by BLIP, then used to search the knowledge base
- **Source transparency** — each response cites the exact textbook sources used
- **No installation required for users** — runs as a live Gradio web app

## ⚙️ Technical Specifications

| Component | Detail |
|---|---|
| Embedding model | `all-MiniLM-L6-v2` (384-dim, sentence-transformers) |
| Vector index | `faiss.IndexFlatIP` with L2-normalized vectors (cosine similarity) |
| Text chunking | LangChain `RecursiveCharacterTextSplitter` — chunk_size=500, overlap=100 |
| Image captioning | `Salesforce/blip-image-captioning-base` |
| Text generation | `llama-3.1-8b-instant` via Groq API |
| Vision/image model | `meta-llama/llama-4-scout-17b-16e-instruct` via Groq API |
| Text retrieval | top-k = 6 chunks |
| Image retrieval | top-k = 4 images, score threshold = 0.20 |
| Query expansion | 3 anatomy-tuned variants generated per image query |
| Context window cap | 3500 characters per query |
| Avg. response time | ~1.2 seconds |
| UI framework | Gradio |

## 🚀 Setup and Installation

### Prerequisites

- Python 3.8 or above
- A valid [Groq API key](https://console.groq.com/)
- Anatomy textbook PDFs (see Knowledge Base section)

### 1. Clone the repository

```bash
git clone https://github.com/SivaPrakash-2715/AnatoTutor.git
cd AnatoTutor
```

### 2. Install dependencies

```bash
pip install pymupdf sentence-transformers faiss-cpu numpy pillow \
            transformers gradio groq langchain-text-splitters
```

> If you have a CUDA-enabled GPU, replace `faiss-cpu` with `faiss-gpu` for faster indexing.

### 3. Set your Groq API key

Open `src/rag/llm_generator.py` and replace the placeholder with your key:

```python
GROQ_API_KEY = "your_groq_api_key_here"
```

Or set it as an environment variable and update the code to read from `os.environ`.

### 4. Configure data paths

The scripts default to `E:\Anato-tutor\` (Windows dev path). Update the `BASE` path constant at the top of each script to match your local directory:

```python
BASE = "/path/to/your/AnatoTutor"  # e.g., BASE = os.path.dirname(os.path.dirname(...))
```

### 5. Run the setup pipeline

Execute these scripts **in order**, once:

```bash
# Step 1 — Extract and clean text from PDFs
python src/parsing/smart_parse.py

# Step 2 — Extract images from PDFs
python src/parsing/extract_images.py

# Step 3 — Generate captions for all images (downloads BLIP on first run)
python src/preprocessing/caption_images.py

# Step 4 — Build text FAISS index
python src/embeddings/build_multimodal_index.py

# Step 5 — Build image FAISS index
python src/embeddings/build_image_index.py
```

### 6. Launch the application

```bash
python src/ui/app.py
```

The Gradio interface will start and print a local URL (e.g., `http://127.0.0.1:7860`). Open it in any browser.

## 🖥️ Using AnatoTutor

### Text Query Tab

1. Type any anatomy question in the input box (e.g., *"Describe the brachial plexus for 10 marks"*)
2. Optionally enter a marks value to calibrate response depth
3. Click **Ask AnatoTutor** or use a Quick Question button
4. View the structured answer, related images, and textbook sources across three output tabs

### Image Query Tab

1. Upload an anatomy image — X-ray, diagram, specimen photo, or textbook figure (JPG/PNG)
2. Optionally add a specific question about the image
3. Click **Analyse Image**
4. The system displays an image interpretation confirmation, then generates a full structured response

### Marks-Based Response Depth

| Marks | Response |
|---|---|
| 2 marks | Brief definition and key point |
| 4–5 marks | Definition + Structure + Function |
| 10 marks | Full structured response with all six sections |
| Not specified | Balanced medium-length response |

## 📤 Output Format

Every text response is organized into six labeled sections:

| Section | Content |
|---|---|
| 📖 Definition | Concise definition of the anatomical structure |
| 🦴 Structure | Location, dimensions, subdivisions, and relationships |
| ⚙️ Function | Physiological role and purpose |
| 🏥 Clinical Relevance | Associated conditions, injuries, or pathologies |
| 🎯 Key Points | Bullet-point exam-ready summary |
| 💡 Memory Tip | Mnemonic or memorable phrase for recall |

## ⚠️ Limitations

- **Image retrieval is caption-based, not visual:** AnatoTutor does not directly reason about pixel content. It converts uploaded images into text captions (via BLIP) and retrieves textbook content using those captions. True visual understanding (e.g., identifying subtle pathological features) is outside the current system's scope.
- **Knowledge base is fixed:** The system answers only from ingested textbook content. Queries outside the scope of the five source books may return limited results.
- **Not for clinical use:** AnatoTutor is designed exclusively for educational purposes and must not be used for medical diagnosis or clinical decision-making.
